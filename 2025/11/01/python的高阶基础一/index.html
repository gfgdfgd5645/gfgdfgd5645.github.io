设计模式

设计模式分为单例模式和工厂模式
设计模式是一种编程套路，可以极大的方便程序的开发。最常见，最经典的设计模式，就是我们所学习的面向对象了
除了面向对象外，在编程中也有很多既定的套路可以方便开发，我们称之为设计模式：

  • 单例模式，工厂模式等
  • 建造者，责任链，状态等
    设计模式非常多，在这里主要讲解单例模式和工厂模式
  • 单例模式 创建类的实例后，就可以得到一个完整的，独立的类对象，某些场景下我们需要一个类无论获取多少次类对象，都仅仅提供一个具体的实例，用以节省类对象的开销和内存开销，实现方式有很多种 这里的s1和s2是两个不同的实例，但是他们都指向了同一个类对象，所以在内存中只存在一个类对象，节省了内存开销
  • 工厂模式 当需要大量创建同一类型的对象时，可以使用工厂模式，即，从原生的使用类的构造方法去创建对象的形式，迁移到，基于工厂提供的方式创建对象的形式
    传统的形式为 工厂式生产对象 使用工厂类的da_create()方法去创建具体的类对象 有以下优点
  1. 大批量的创建对象时有统一的入口，易于代码的维护
  2. 当发生修改时，仅修改工厂类的创建方法即可

多线程编程

本章有两个重要的概念 即线程和进程
进程： 就是一个程序运行在系统上，那么便称作为这个程序为一个运行进程，并分配id方便系统进行管理
线程： 一个进程中可以包含多个线程，线程是进程的子集，一个进程至少有一个线程
进程之间内存是隔离的 线程之间共享内存
一般来说，程序的运行是由顺序的，但是多线程就可以让程序同时运行，增加运行效率
每个独立的进程有一个程序运行的入口、顺序执行序列和程序的出口，但是线程不能够独立执行，必须依存在应用程序中，由应用程序提供多个线程执行控制。每个线程都有他自己的一组CPU寄存器，称为线程的上下文，该上下文反映了线程上次运行该线程的CPU寄存器的状态。指令指针和堆栈指针寄存器是线程上下文中两个最重要的寄存器，线程总是在进程得到上下文中运行的，这些地址都用于标志拥有线程的进程地址空间中的内存。

  • 线程可以被抢占（中断）。
  • 在其他线程正在运行时，线程可以暂时搁置（也称为睡眠） – 这就是线程的退让。
    Python中使用线程有两种方式：函数或者用类来包装线程对象。
    python中一般使用函数和类来包装多线程对象
    python的多线程可以通过threading模块来实现 这里的多进程就是通过创建threading.Thread对象来实现的，其还有很多参数如下
    ob = threading.Thread(group,target,name,args,kwargs,group)
  • target 线程执行的函数
  • name 线程的名称
  • args 线程执行函数的参数 元组的形式 传递元组时需要注意就算没有第二个参数也要写逗号
  • kwargs 线程执行函数的关键字参数 字典的形式
  • group 线程组
    想让其运行通过实例化的对象.star来实现
    传参由两种，一种是args，通过元组传参 一种kwargs，通过字典来传参
    例如
    kn = threading.Thread(target=hello,args=(“hello world”,))
    km = threading.Thread(target=hello,kwargs={“name”:”hello world”})
