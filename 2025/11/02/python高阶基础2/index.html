网络编程

socket简称套接字，是计算机网络通信的基本单位
在一般的模式中，一般都会存在服务端和客户端两种角色

  • 客户端 主动与服务端建立连接，发送请求，等待服务端的响应
  • 服务端 被动等待客户端的请求，并处理请求，发送响应
    服务端编程主要分为几个步骤
  1. 创建socket对象
    import socket
    s = socket.socket()
  2. 绑定ip和端口
    s.bind(hpst, port)
  3. 监听
    s.listen(backlog)
    这里设置的backlog是整数，表示监听队列的长度，如果超出队列长度，会报错,可以不填，会自动设置一个合理的值
  4. 接受连接
    a,y = s.accept()#分别为连接对象，客户端地址信息
    print(f”客户端的ip为{a}和端口{y}”)
    accept方法是堵塞方法，如果没有连接，会卡住，直到有连接
    accept返回的是一个二元元组，使用上述形式，用两个变量接收二元元组的两个元组
  5. 客户端连接后，通过recv方法，接收客户端发送的数据
    while True:
    data = a.recv(1024).decode(“utf-8”)
    #recv方法的返回值是字节数组bytes，需要decode解码成字符串
    #recv的方法的传参是buffsize，缓冲区大小，一般设置为1024即可
    if data == “exit”:
    break
    print(f”客户端发送的数据为{data}”)

#可以通过while true 无限循环来持续接收客户端发送的数据
6. 客户端发送数据后，通过send方法，发送数据给客户端
while True:
data = conn.recv(1024).decode(“utf-8”)
if data == “exit”:
break
print(f”客户端发送的数据为{data}”)
conn.send(data.encode(“utf-8”))
7. conn.close()#关闭连接
s.close()#关闭服务端

正则表达式

正则表达式。又称规则表达式，是使用单个字符串来描述，匹配某个语句规则的字符串，常被用来检索，替换换那些符合那个模式（规则）的文本
简单来说，正则表达式就是使用：字符串定义规则，并通过规则去验证字符串是否匹配
比如，验证一个字符串是否是符合条件的电子邮件地址，只需要配置好正则规则，即可匹配任意邮箱
python正则表达式，使用re模块，并基于re模块中三个基础模块来做正则匹配
分别是match，search,findall

  • re.match 匹配开头
    从被匹配的字符串开头进行匹配，匹配成功返回匹配对象（包含匹配的信息），匹配不成功返回空
  • re.serch 全局匹配，匹配第一个命中项
  • re.findall 全局匹配，匹配全部命中项

异步编程

在这里有几个概念需要理清 协程 事件循环 任务

  • 协程是可以暂停运行和恢复运行的函数，通过携程函数async定义 当执行时，不会真的执行，而是返回一个协程对象，在执行协程对象时才会执行里面的代码
    协程编程最大的特点就是协程可以恢复也可以暂停 当出现await就会暂停运行，让出控制权，等到await后的完成了，再继续申请控制权
  • 事件循环 用来调度协程运行 主要三大功能 1.检查携程 检查有没有可以运行的协程 2.让出控制 将控制权传递可以运行的进程 3.等待协程 等当前协程完成将控制权传递给自己 这是一个协程
  • 任务 任务是对协程的封装 除了协程本身还包括了协程的状态 只要一个协程被包装成任务了就会被事件循环调度
    在这里可以简单理解为这一切就是一个摩天轮 中间的就是时间循环 摩天轮里的每个包厢就是任务 每个包厢里面的就是协程 想要被调度就必须被包装成任务
    所以顺序为
  1. 定义协程函数
  2. 包装协程为任务
  3. 建立事件循环

对于任务循环，需要有两大概念需要认识，crooutine和task
在python语境中提到coroutine，一种是coroutine function或者是coroutine object

  • coroutine function 协程函数 通过async定义的函数 也就是说当执行这个函数时，不会真的执行，而是返回一个协程对象也就是coroutine object
  • coroutine object 协程对象 通过async定义的函数返回的对象
    如果想要执行协程函数，有几大步骤
  1. 进入asyncio的模式，也就是进入事件循环
    正常写的是synchronize模式，切换到ayynchronize,基本上我们只用一个入口函数asyncio.run() 参数是coroutine
    这里的入口函数asyncio.run() 第一会自动创建一个事件循环，第二会把coroutine变成任务循环里的第一个task
  2. 创建协程对象

增加task在任务循环

在这里回忆一下前面说到的将coroutine function调用后不会运行 返回的是一个coroutine object，由此可得await后是一个object
await一个object后发生以下几件事
1.这个coroutine object被包装成一个task，并告知任务循环
2.并且告诉任务循环，需要等到say_after运行完毕，才能继续运行，建立了一个依赖关系
3.yield出去，将控制权交给任务循环，任务循环运行其他task
4.完成后，await会将coroutine里面的返回值返回

上述代码运行过程应该是
1. asyncio.run将main函数包装成一个task，并告知任务循环
2. 任务循环中只有一个main的task，所以运行main
3. 运行到 await say_after(1, ‘hello’) 将say_after这个调用函数完变成的coroutine object包装成一个task，并告知任务循环，并建立了需要等到say_after运行完毕，才能继续运行，建立了一个依赖关系
4. 然后yield出去，将控制权交给任务循环，任务循环运行其他task，现在任务循环里有两个task，但是main和say_after有一个依赖关系，所以继续执行say_after，执行时发现里面也有一个await，这里的asyncio.sleep其实也是一个coroutine,也将这个函数变成task加入任务循环并且yield出去
5. 目前任务循环内的task有main，say_after,sleep
任务循环内的依赖关系应该是 main函数里面await say_after say_after里面await asyncio.sleep，所以先执行asyncio.sleep，然后执行say_after，最后执行main,这就已经执行完了第一个say_after，然后就是第二个，同样的步骤
这里可以看到，这些控制权的交替都是显形的，任务循环不能去强控制权，必须是task对象自己yield出去，或者这个task对象运行完毕，然后任务循环才能继续运行其他task
这里的有一个问题，要是这么写，执行是顺次的，需要等待时间并不比没有asyncio少实践，为啥不能让两个函数的时间压在一起，也就是两个协程同时运行，然后在await的时候，让出控制权，等到一个完成了，再继续运行，这样就节省了时间 这些应该是我们直接用awit把coroutine变成task会遇到的问题，因为后面的coroutine必须要等到前面的coroutine运行完毕，所以他们才没能同时等待
asyncio里面有一个asyncio.create_task()可以帮我们解决这个问题

这里的区别是，我们使用asyncio.create_task()将两个协程变成task，并且告知当前创建的task已经是就绪状态，但是现在控制权还是在main手里，所以会继续创建下一个task 之后才到await task1 和await task2
这里的await task1 和await task2 其实是告诉任务循环，需要等到task1和task2运行完毕，才能继续运行，建立了一个依赖关系，这里也可以接收return，也是在awit前面

接收函数时，一次一次的接受 有点慢，可以使用asyncio.gather() ret = await asynvio.gather(task1, task2)
这里的ret是task1和task2的返回值，可以使用ret[0]和ret[1]来获取
gather并不是一个coroutine,但是他会返回一个叫做future的东西，future也是可以用await的，它的参数是coroutine或者是task甚至可以是他自己本事future
如果是coroutine，首先会把coroutine变成task，注册到任务循环里，然会返回future值，当我们await future时，就相当于告诉任务循环，需要等到future里的task全部执行完毕才能继续执行，执行完毕后程序的返回值就是future的返回值会通过await以list的形式返回，里面的顺序也是跟future的顺序一致

迭代器与生成器

我们知道python里有些对象是可以for循环的，有些是不可以的，具体为什么可以for循环，就得知道for循环的原理 在python中能使用for循环的对象叫做可迭代对象，也叫做iterables iterables会包含__iter方法
for循环遍历的原理其实是 先调用iterables里的__iter__方法，获取一个迭代器，然后再调用迭代器的__next__方法，获取下一个元素，有下一个元素就返回下一个元素，没有事则跳出循环
可迭代对象的__iter__会返回一个迭代器，迭代器的__iter__会返回自己本身
迭代器是一个可以记住遍历的位置的对象。
迭代器对象从集合的第一个元素开始访问，直到所有的元素被访问完结束。迭代器只能往前不会后退。
迭代器有两个基本的方法：iter() 和 next()。
字符串，列表或元组对象都可用于创建迭代器：

也可以直接使用for循环来遍历

迭代器在类中的实现需要定义初始化方法，分别是__iter__和__next__

StopIteration
StopIteration 异常用于标识迭代的完成，防止出现无限循环的情况，在 next() 方法中我们可以设置在完成指定循环次数后触发 StopIteration 异常来结束迭代。 需要注意的是__next__方法是只能往下走的

生成器
在 Python 中，使用了 yield 的函数被称为生成器（generator）。yield 是一个关键字，用于定义生成器函数，生成器函数是一种特殊的函数，可以在迭代过程中逐步产生值，而不是一次性返回所有结果。定义生成器有两种方法分别是生成器函数和生成器表达式,每次运行到yield，就会在yield返回一个生成的值，下次运行时，就会从上次返回的yield处继续，每次调用生成器都需要使用next方法
