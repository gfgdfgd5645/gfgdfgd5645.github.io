docker

docker的基本概念

运行容器的主机叫做宿主机
docker与虚拟机最大区别就是docker容器与宿主机共用一个系统内核，而虚拟机都完整包含了一个系统的内核
镜像是容器的模板，docker仓库就是存储分享的地方
docker的命令可以大概分为下载，运行，挂载卷，run其它参数，dokcer网络，docker-compose

docker原理基本概念
docker使用了linux的两大原生功能

  • cgroups 用来限制容器的资源使用 可以设定每个进程的cpu，内存，网络等资源的使用上限
  • namespace 用来隔离容器的进程的资源视图，使得进程只能看到自己内部的进程id和网络资源和文件目录而看不到宿主机的
    所以容器就是一个特殊的进程，但是当我们进入容器内部时，就像是一个独立的linux系统，都是一个独立的运行环境，但是其内部的linux是极度简洁的 很多系统工具都是缺失的
    docker exec xxxxxx(容器id) linux命令此命令可以进入容器内部执行linux命令
    或者docker exec -it xxxxxx(容器id) /bin/sh 进入容器内执行linux命令 这里的/bin/sh是linux的shell 可以使用bash替换

docker下载

docker pull docker.io/library/naginx:latest
docker pull用来从仓库下载镜像

  • docker.io为docker仓库的注册地址 registry注册表
  • library为命名空间 library是docker的官方命名空间 如果是官方命名空间那可以省略
  • nginx为镜像名称 上面三者合起来就是repositrory镜像库
  • latest为镜像的版本号 如果不指定版本号，默认是latest
    其实还有一个特殊的后缀 –platform=xxx 拉取特定架构的镜像 但是docker会自动匹配与当前架构匹配的镜像

其实可以直接输入docker run nginx docker识别到本地没有这个镜像 就会去仓库下载
docker images 查看本地镜像
docker search 搜索镜像
docker rmi 删除镜像

docker的运行

下面的参数如果不特殊说明都是docker run的命令
docker run naginx
docker run后面可以接一个镜像的名字或者id 这一步就是使用镜像创建一个容器并运行
如果要查看docker实例运行状态 可以使用docker ps命令查看 ps指的是process status 是从linux命令中继承过来
在输出的命令中能看到容器是有name的 如果不手动指定系统会自动分配

  • -d
    如果是按照开头的命令启动docker 那么容器的运行是堵塞性质的 会占用窗口 如果要后台运行 需要加-d参数 这里的d指的是 detached mode 也就是分离模式

  • rm
    删除容器直接rm+容器id 或者rm+容器name 正在运行的容器需要加-f参数 强制删除 这里的f指的是force 强制

  • -p
    这里的-p指的是端口映射，每个docker容器都运行在一个独立的虚拟环境里，宿主机并不能直接访问到docker实例网络环境
    这个时候我们需要添加-p参数来指定端口映射 格式为-p 宿主机端口:容器端口 顺序是先外后内

  • -v(绑定挂载)
    与-p类似，-v则是把宿主机与容器的文件目录进行绑定，容器内对文件夹的修改会影响宿主机的文件夹 这种目录也称为挂载卷
    这种挂载卷有一个好处，就是当我们删除容器时，容器目录都会被删除，但是挂载卷不会因为容器的删除而消失
    绑定完成后，宿主机的文件就会覆盖到容器的文件
    格式为-v 宿主机目录:容器目录 顺序是先外后内

  • -v(命名挂载)
    命名挂载是创建一个存储空间(命名卷) 然后把存储空间挂载到容器中
    格式为-v 存储空间名称:容器目录 顺序是先外后内
    首先先创建一个挂载卷
    docker volume create 存储空间名称
    然后再使用-v参数挂载
    docker run -d -p 80:80 -v 存储空间名称:/usr/share/nginx/html nginx
    如果要查询挂载对应的宿主机的真实目录可以使用docker volume inspect 存储空间名称
    docker volume list 查看所有挂载卷
    docker volume rm 存储空间名称 删除挂载
    docker volume prune -a 删除所有未使用的挂载卷

  • -e 设置环境变量
    例如我们要使用一个mangodb的数据库 在创建时需要传入账号和密码 在这里就可以作为环境变量传入
    docker run -d -p 27017:27017 -e MONGO_INITDB_ROOT_USERNAME=root -e MONGO_INITDB_ROOT_PASSWORD= mongo

  • –name 指定容器名称
    docker run -d -p 80:80 –name nginx nginx
    名字必须是唯一的，没有也为所谓，如果不输入名字，docker会自动分配一个随机名称，功能与id一样

  • -it
    -it代表的是将容器的终端绑定到当前终端

  • –rm代表的是，当容器停止的时候就将当前容器删除掉

  • –restart
    这个参数用来配置容器在停止时的重启策略，例如 –restart always 代表的是容器停止后会自动重启 用来处理崩溃等问题
    还有参数与restart always相似就是–restart unless-stopped 代表的是容器停止后会自动重启，但是如果容器手动停止了，则不会重启

docker的调试

docker的启动与停止

  • docker start 容器id
  • docker stop 容器id
  • docker restart 容器id
    docker会记录参数的写入 每一次重启不需要重新写入参数 如果忘记了参数可以使用docker inspect 容器id 查看之前写入的参数

docker的状态
docker ps 会显示所有正在运行的容器
docker ps -a 会显示所有容器
docker ps -q 只显示容器id

docker create 镜像名称

创建容器

dockerfile
前面提到，容器是通过镜像创建的，而镜像是通过dockerfile创建的

  1. 先创建一个Dockerfile文件
    需要选择基础镜像 在这里可以选择需要的依赖的环境
    指令为 from python:3.7-alpine

workdir /app 指定工作目录 之后的所有操作都在这个目录下

copy .. 这两个点都代表的是当前目录 第一个点是宿主机与dockerfile同目录的文件 第二点代表的是现在所处的工作目录

run pip install flask run指令后代表着需要在容器中执行的命令

expose 80 声明使用的端口 但只是声明 不是强制使用

CMD [“python”, “app.py”] 指定容器启动时执行的命令 ，一个dockerfile只能有一个CMD指令
与cmd命令类似的是entrypoint 后者优先级更高，不容易被覆盖

  1. 创建镜像
    docker build -t 镜像名称:版本号 . 这个点代表的是在当前文件夹构建
    在同一目录下直接使用docker run 镜像名称 就可以运行刚刚创建的容器

3.推送镜像
如果想推送到docker仓库 需要先登录

docker login
这里需要注意如果想用推送到仓库，需要在创建的docker镜像前面加上自己的命名空间
docker push 命名空间/镜像名称:版本号

docker网路

docker网络默认为bridage 也就是桥接网络，所有的容器都默认连接到这个网络中，每个容器都分配了一个内部的ip地址，在这个内网里面，容器容器可以利用内部IP地址互相访问，但宿主机和容器之间的网络是隔离的
bridage网络
创建子网
–network network1 指定某个容器使用的子网
docker network create network1
同子网的容器可以互相访问 但是不同子网的容器是不能互相访问的，创建子网还有一个好处，同一子网的容器可以互相访问，可以使用容器的名字互相访问，而不必使用ip地址这主要依赖于docker内部的特殊dns

host网络
在host网络中，docker容器直接共享宿主机的网络，无需端口映射，容器可以直接使用宿主机的ip地址访问 –network host

none网络
none网络是没有网络的，容器只能使用localhost访问，不能访问宿主机

docker compose
一个完整的应用可能是有多个部分组成的，但是如果每个容器都单独配置，略显麻烦，docker compose就是用来解决这个问题的
docker compose使用yaml文件来配置，里面列出了docker是怎么创建的，以及如何协同工作的，我们可以简单的将docker-compose理解为多个docker run命令
其结构为

最顶级的是server元素，server元素下可以有多个service元素，每个service元素代表一个容器，容器的name就是dockerpose的server名称，镜像也就是image，端口映射就是ports，挂载卷就是volumes，环境变量就是environment
大概样子是

唯一与docker run命令不同的是，在调用docker run时需要指定网络参数，但docker compose不需要，因为docker compose创建时会自动创建一个子网，并将所有docker-compose创建的容器连接到同一个子网中

docker还有一个特殊功能，就是自定义启动顺序
